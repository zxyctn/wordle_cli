From domain/ports.py (being created in parallel):
```python
from abc import ABC, abstractmethod

class WordRepository(ABC):
    @abstractmethod
    def get_valid_words(self) -> list[str]:
        """Return list of valid 5-letter words."""
        pass
    
    @abstractmethod
    def is_valid_word(self, word: str) -> bool:
        """Check if word exists in dictionary."""
        pass
    
    @abstractmethod
    def get_random_target(self) -> str:
        """Select a random word as the target."""
        pass
```

- `__init__(self, seed: int | None = None)`: 
  - Store game_words list as private `_words` attribute
  - Convert to set for O(1) validation lookups as `_word_set`
  - Initialize random.Random(seed) as `_random` for reproducible testing

- `get_random_target() -> str`: Return `self._random.choice(self._words)`

Add docstring explaining this adapter loads words from words.py module and uses in-memory storage with seed support for testing.
